# 數學模型設計工作流程指引

本指引涵蓋老虎機數學模型設計的完整工作流程，包括 Paytable 設計、Reel Strip 配置、RTP 計算、Volatility 調校、Hit Frequency 計算與獎勵功能設計。

---

## 1. Paytable 設計

### 1.1 符號組合定義

Paytable（賠率表）定義了每種符號組合對應的獎金倍數。設計時需考慮以下要素：

**符號分類**

| 類別 | 說明 | 範例 |
|------|------|------|
| 高賠付符號 | 出現頻率低、獎金倍數高 | 7、鑽石、皇冠 |
| 中賠付符號 | 出現頻率與獎金倍數適中 | 鈴鐺、星星、櫻桃 |
| 低賠付符號 | 出現頻率高、獎金倍數低 | A、K、Q、J、10 |
| 特殊符號 | 具備特殊功能 | Wild（百搭）、Scatter（散佈） |

**符號組合規則**

- 定義每條 Payline 上需要匹配的最少符號數量（通常為 3 個）
- 設定從左至右（Left-to-Right）或雙向（Both Ways）的匹配方向
- Wild 符號可替代除 Scatter 外的所有符號
- Scatter 符號不需在 Payline 上，出現在任意位置即可觸發

### 1.2 獎金倍數設定

獎金倍數以投注金額的倍數表示。典型的 5×3 捲軸老虎機倍數範例：

| 符號 | 3 個匹配 | 4 個匹配 | 5 個匹配 |
|------|----------|----------|----------|
| 高賠付 A | 50x | 200x | 1000x |
| 高賠付 B | 30x | 100x | 500x |
| 中賠付 A | 15x | 50x | 200x |
| 中賠付 B | 10x | 30x | 100x |
| 低賠付 A | 5x | 15x | 50x |
| 低賠付 B | 3x | 10x | 30x |

**設定原則**

- 最高獎金倍數應與目標 Volatility 一致
- 高賠付符號的 5 個匹配倍數通常為 3 個匹配的 10–20 倍
- 確保各符號間的倍數梯度合理，避免跳躍過大

### 1.3 RTP 驗算方法

RTP（Return to Player）為長期統計平均回報百分比，業界標準範圍為 **94%–98%**。

**單一符號組合的 RTP 貢獻計算**

```
符號組合 RTP 貢獻 = 出現機率 × 獎金倍數
```

**整體基礎遊戲 RTP 計算**

```
基礎遊戲 RTP = Σ（每種獲勝組合的出現機率 × 對應獎金倍數）
```

**完整 RTP 計算（含獎勵功能）**

```
總 RTP = 基礎遊戲 RTP + Free Spin RTP 貢獻 + Bonus Round RTP 貢獻 + Multiplier RTP 貢獻 + Progressive Jackpot RTP 貢獻
```

**RTP 驗算步驟**

1. 列出所有可能的符號組合及其出現機率
2. 計算每種組合的 RTP 貢獻
3. 加總所有組合的 RTP 貢獻得到基礎遊戲 RTP
4. 加入各獎勵功能的 RTP 貢獻
5. 驗證總 RTP 落在 94%–98% 範圍內
6. 若 RTP 低於 92%，需特別注意——某些市場禁止 RTP 低於此門檻

**RTP 驗算注意事項**

- 建議使用蒙地卡羅模擬（Monte Carlo Simulation）進行至少 1,000 萬次旋轉的統計驗證
- 模擬結果與理論計算的誤差應小於 0.5%
- 記錄每次調整的 RTP 變化，建立版本追蹤

## 2. Reel Strip 配置

### 2.1 Virtual Reel 加權映射系統

Virtual Reel（虛擬捲軸）是現代老虎機的核心機制，透過加權映射系統控制每個符號的出現機率，使其獨立於實體捲軸上的符號排列。

**運作原理**

1. RNG 產生一個隨機數值（例如 0–127）
2. 將該數值對應到 Virtual Reel 的權重表
3. 權重表決定最終顯示的符號

**Virtual Reel 權重表範例（單一捲軸）**

| 符號 | 權重 | 機率 |
|------|------|------|
| 7（高賠付） | 2 | 2/128 = 1.56% |
| 鑽石（高賠付） | 4 | 4/128 = 3.13% |
| 鈴鐺（中賠付） | 10 | 10/128 = 7.81% |
| 星星（中賠付） | 12 | 12/128 = 9.38% |
| A（低賠付） | 20 | 20/128 = 15.63% |
| K（低賠付） | 20 | 20/128 = 15.63% |
| Q（低賠付） | 20 | 20/128 = 15.63% |
| J（低賠付） | 20 | 20/128 = 15.63% |
| 10（低賠付） | 18 | 18/128 = 14.06% |
| Wild | 2 | 2/128 = 1.56% |
| **總計** | **128** | **100%** |

**映射函數虛擬碼**

```typescript
function mapToSymbol(rngValue: number, virtualReel: VirtualReelEntry[]): string {
  let cumulative = 0;
  for (const entry of virtualReel) {
    cumulative += entry.weight;
    if (rngValue < cumulative) {
      return entry.symbol;
    }
  }
  // 安全回退：返回最後一個符號
  return virtualReel[virtualReel.length - 1].symbol;
}
```

### 2.2 符號權重分配方法

**分配原則**

- **高賠付符號**：權重最低（1–5），確保稀有性
- **中賠付符號**：權重適中（8–15），維持適度出現頻率
- **低賠付符號**：權重最高（15–25），提供基礎命中率
- **特殊符號（Wild/Scatter）**：權重極低（1–3），控制觸發頻率

**權重調整策略**

1. 先設定目標 RTP 與 Hit Frequency
2. 從均勻分佈開始，逐步調整各符號權重
3. 每次調整後重新計算 RTP，確保仍在目標範圍內
4. 使用模擬驗證調整結果

**多捲軸配置注意事項**

- 每個捲軸可以有不同的權重配置
- 中間捲軸（如第 3 軸）通常降低高賠付符號權重，增加遊戲難度
- 確保 Virtual Reel 總權重不為零
- 建議總權重使用 2 的冪次（如 64、128、256），便於 RNG 映射

## 3. Hit Frequency 計算

### 3.1 計算公式

Hit Frequency（命中頻率）表示任一次旋轉產生獲勝組合的機率百分比：

```
Hit Frequency = (總獲勝組合數 ÷ 總可能組合數) × 100
```

**範例計算**

假設一台 5×3 捲軸老虎機，每個捲軸有 128 個虛擬位置：

- 總可能組合數 = 128 × 128 × 128 × 128 × 128 = 34,359,738,368
- 若總獲勝組合數 = 10,307,921,510
- Hit Frequency = (10,307,921,510 ÷ 34,359,738,368) × 100 ≈ 30%

### 3.2 計算步驟

1. **計算總可能組合數**：各捲軸的虛擬位置數相乘
   ```
   總可能組合數 = Reel1_positions × Reel2_positions × ... × ReelN_positions
   ```

2. **列舉所有獲勝組合**：遍歷所有 Payline，計算每條線上每種獲勝符號組合的出現次數

3. **去除重複計算**：若同一次旋轉可在多條 Payline 上獲勝，需注意不重複計算旋轉次數（但獎金可疊加）

4. **套用公式**：將總獲勝組合數除以總可能組合數，乘以 100

### 3.3 目標範圍建議

| Hit Frequency 範圍 | 玩家體驗 | 適用場景 |
|---------------------|----------|----------|
| 15%–25% | 獲勝間隔較長，適合高波動遊戲 | 追求大獎的玩家 |
| 25%–35% | 適中的獲勝頻率 | 大多數標準老虎機 |
| 35%–45% | 頻繁獲勝，單次獎金較小 | 休閒玩家、低波動遊戲 |

**注意事項**

- Hit Frequency 計算結果必須在 0–100 之間（百分比值）
- 若 Hit Frequency 低於 10% 或高於 60%，應重新檢視符號權重分佈
- Hit Frequency 與 Volatility 密切相關，調整時需同步考量

## 4. Volatility 調校

### 4.1 波動性等級定義

Volatility（波動性）衡量老虎機的風險等級，決定獎金大小與頻率的分佈。

| 等級 | 特徵 | Hit Frequency | 最高獎金倍數 | 適合玩家 |
|------|------|---------------|-------------|----------|
| 高波動 | 大獎低頻 | 15%–25% | 5,000x–50,000x | 追求大獎、能承受長期無獎的玩家 |
| 中波動 | 平衡分佈 | 25%–35% | 1,000x–5,000x | 大多數玩家 |
| 低波動 | 小獎高頻 | 35%–45% | 100x–1,000x | 休閒玩家、偏好穩定回報的玩家 |

### 4.2 符號權重分佈策略

**高波動策略**

- 高賠付符號權重極低（1–2），使大獎極為稀有
- 低賠付符號權重極高（25–30），填充大部分結果
- Wild 符號權重極低（1），減少替代機會
- 獎金倍數梯度陡峭：低賠付與高賠付之間差距大

```
權重分佈範例（總權重 128）：
高賠付符號：1–2（佔 1%–2%）
中賠付符號：5–8（佔 4%–6%）
低賠付符號：25–30（佔 20%–23%）
Wild：1（佔 0.8%）
```

**低波動策略**

- 高賠付符號權重適度提高（4–6），增加大獎出現機會
- 低賠付符號權重適中（15–18），降低空轉比例
- Wild 符號權重提高（3–5），增加替代獲勝機會
- 獎金倍數梯度平緩：各等級符號倍數差距較小

```
權重分佈範例（總權重 128）：
高賠付符號：4–6（佔 3%–5%）
中賠付符號：12–15（佔 9%–12%）
低賠付符號：15–18（佔 12%–14%）
Wild：3–5（佔 2%–4%）
```

**中波動策略**

- 各符號權重介於高波動與低波動之間
- 獎金倍數梯度適中
- 提供均衡的遊戲體驗

```
權重分佈範例（總權重 128）：
高賠付符號：2–4（佔 2%–3%）
中賠付符號：8–12（佔 6%–9%）
低賠付符號：18–22（佔 14%–17%）
Wild：2–3（佔 1.5%–2.3%）
```

### 4.3 Volatility 調校流程

1. 確定目標波動性等級
2. 依據上述策略設定初始符號權重
3. 計算 RTP，確保在 94%–98% 範圍內
4. 計算 Hit Frequency，確認符合目標範圍
5. 執行蒙地卡羅模擬（至少 1,000 萬次旋轉）
6. 分析獎金分佈曲線，確認符合目標波動性
7. 微調權重直到所有指標達標

## 5. 獎勵功能設計

### 5.1 RTP 貢獻計算總則

每種獎勵功能對整體 RTP 的貢獻獨立計算，最終加總：

```
總 RTP = 基礎遊戲 RTP + Free Spin RTP 貢獻 + Bonus Round RTP 貢獻
       + Multiplier RTP 貢獻 + Progressive Jackpot RTP 貢獻
```

### 5.2 Free Spin（免費旋轉）

**觸發條件**：通常由 3 個以上 Scatter 符號觸發

**RTP 貢獻計算**

```
Free Spin RTP 貢獻 = 觸發機率 × 平均免費旋轉次數 × 每次免費旋轉的平均獎金 ÷ 觸發投注金額
```

**設計要點**

- 免費旋轉期間可使用不同的 Reel Strip 配置（通常提高高賠付符號權重）
- 免費旋轉可重新觸發（Retrigger），需納入 RTP 計算
- 典型 Free Spin RTP 貢獻：5%–15%

### 5.3 Bonus Round（獎勵回合）

**觸發條件**：由特定符號組合或 Scatter 觸發

**RTP 貢獻計算**

```
Bonus Round RTP 貢獻 = 觸發機率 × 平均獎勵回合獎金 ÷ 觸發投注金額
```

**常見類型**

| 類型 | 說明 | 典型 RTP 貢獻 |
|------|------|---------------|
| Pick-and-Click | 玩家選擇物件揭示獎金 | 2%–5% |
| Wheel of Fortune | 轉盤決定獎金倍數 | 1%–3% |
| Trail Bonus | 沿路徑前進收集獎金 | 2%–4% |
| Cascading Wins | 獲勝符號消除後新符號落下 | 3%–8% |

### 5.4 Multiplier（乘數）

**觸發條件**：隨機觸發或由特定符號啟動

**RTP 貢獻計算**

```
Multiplier RTP 貢獻 = 觸發機率 × (平均乘數值 - 1) × 觸發時的平均基礎獎金 ÷ 投注金額
```

**設計要點**

- 乘數可套用於單線獎金或整次旋轉獎金
- 常見乘數值：2x、3x、5x、10x
- 乘數可與 Free Spin 疊加，大幅提升波動性

### 5.5 Progressive Jackpot（累積獎金）

**運作機制**：每次投注抽取固定比例注入獎金池

**RTP 貢獻計算**

```
Progressive Jackpot RTP 貢獻 = 每次投注的抽取比例（通常 0.5%–2%）
```

**設計要點**

- 抽取比例直接等於 Progressive Jackpot 的 RTP 貢獻
- 可設計多層級獎金池（Mini、Minor、Major、Grand）
- 觸發條件可為隨機觸發或特定符號組合
- 需確保抽取比例加上其他 RTP 貢獻後，總 RTP 仍在目標範圍內

### 5.6 整合計算範例

| 項目 | RTP 貢獻 |
|------|----------|
| 基礎遊戲 | 85.0% |
| Free Spin | 7.5% |
| Bonus Round | 2.5% |
| Multiplier | 1.5% |
| Progressive Jackpot | 1.0% |
| **總 RTP** | **97.5%** |

**驗證要點**

- 確認總 RTP 落在 94%–98% 範圍內
- 各獎勵功能的 RTP 貢獻應獨立驗算
- 使用蒙地卡羅模擬驗證理論計算的準確性
- 記錄各功能的觸發頻率與平均獎金，作為認證文件的一部分
